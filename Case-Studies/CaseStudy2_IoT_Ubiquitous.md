# Case Study 2: IoT and Ubiquitous Computing Based on Cloud

## 1. Introduction

IoT (Internet of Things) refers to interconnected devices that collect and exchange data over the internet.

Ubiquitous Computing refers to embedding computational intelligence into everyday objects.

Both heavily rely on cloud computing for:
- Data storage
- Processing
- Analytics
- Scalability

---

## 2. What is IoT?

IoT is a network of smart devices equipped with:

- Sensors
- Actuators
- Network connectivity
- Unique identifiers

These devices send data to cloud servers for analysis and automation.

---

## 3. IoT Architecture (4 Layers)

1. Device Layer
   - Sensors
   - Smart devices
   - Actuators

2. Gateway Layer
   - Data aggregation
   - Protocol conversion

3. Cloud Layer
   - Data storage
   - Big Data processing
   - AI/ML analytics

4. Application Layer
   - Dashboards
   - Mobile apps
   - Monitoring systems

---

## 4. What is Ubiquitous Computing?

Also called:
- Pervasive Computing
- Ambient Intelligence

It aims to make computing invisible and integrated into daily life.

Examples:
- Smart refrigerators
- Smart thermostats
- Wearable devices
- Smart traffic systems

---

## 5. IoT vs Ubiquitous Computing

| IoT | Ubiquitous |
|-----|------------|
| Focus on connected devices | Focus on intelligent environment |
| Internet-based | Context-aware computing |
| Device-level connectivity | Environment-level intelligence |

## 8. Security Challenges in IoT

- Device authentication issues
- Firmware hacking
- Botnet attacks (e.g., Mirai)
- Data interception

---

## 9. Security Solutions

- Device certificates
- Secure boot
- End-to-end encryption
- Cloud monitoring
- Firmware updates

---

## 10. Role of Cloud in IoT

Cloud provides:
- Scalability
- Big Data processing
- AI model training
- Real-time analytics
- Global access

---

## 11. Conclusion

IoT and Ubiquitous computing depend heavily on cloud infrastructure for scalable data processing, automation, and intelligent decision-making.
